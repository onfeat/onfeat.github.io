有氧减脂，需要的都在这了

By Zhiyong Cai

发表于 2015-03-19

我们最主要的是要搞懂以下几个问题：何时做、为什么、如何做和做多少

有氧运动

意义：有氧运动就是你身体脂肪库的阀门，它能调节你的卡路里平衡，并提高你的健康水平和给你一个看上去更棒的身材。

特点：
不需要很剧烈
不需要你花一整天的时间

调查
伯明翰大学（英国）的科学家们发现，当男运动员在跑步机上练习时，比他们以同样的练习强度在脚踏车上多燃烧30％的脂肪。
结论
作为有氧运动，直立的需要协调性的运动仍然是一个较好的选择
（对于燃脂的最好有氧方式不要忽略户外和需要调到全身的运动。）

最佳的减脂时间

早晨和负重训练之后

早晨：
最好是低强度的有氧运动

负重训练后：（HIIT）
先高强度训练再有氧运动，分泌的生长荷尔蒙会多3倍
当你进行负重训练时，你将首先燃烧所训练区域的脂肪，并且如果接着进行有氧训练，它会使身体其他部位的多余脂肪燃烧来给身体供能，并减少脂肪储存。

最佳的训练强度

最大化地燃烧体脂发生在训练强度大约是你最大心率的70－80％上（最大心率＝220－你的年龄）。

你不妨尝试一下「说话试验」，在你进行有氧运动时，当你开始变得说话困难时，那么此时你的心率应该达到了最大心率的75－80％，这是燃烧脂肪的最佳心率范围。将你训练的强度控制在你开始变得说话困难以内，并且这会使你的体脂燃烧率处于最佳状态。

开始时的前15分钟以最大心率的80％进行训练，然后的15分钟以最大心率的60％进行训练，这比他们以相反的强度方式多燃烧25%的脂肪。（先高强度再低强度）

害怕减肌肉？——频率与时间是关键

为了保持肌肉的不断增加和脂肪的逐渐减少，每星期尝试着做3次有氧的训练方案，并且要确保你一星期的总有氧训练时间不超过2小时。

有氧运动是否一定要持续

你只需要使你总体的有氧训练时间达到标淮，就可以让你燃烧更多的卡路里和脂肪。
如果你已经对所做的有氧训练感到十分厌烦，你仍然应该努力遵循每周至少做3次，每次不少于30分钟的有氧训练。但那并不是说你的半小时有氧训练必须持续进行——你可以在训练中间休息10分钟，这样仍能燃烧相同的脂肪数量。在一台跑步机上进行3次10分钟的跑步训练（最大心率的80％），并在训练中间加入20分钟的休息时间，他们燃烧的卡路里和脂肪与他们在同一强度下做一次连续的30分钟跑步的数量相同。

具体实现方法：HIIT

通常HIIT20分钟的训练比在跑步机上连续跑一个小时还要有效,重点是前者达到了更优秀的效果并且节省了四十分钟之多,这种训练方式会让你在20分钟之内耗尽100%的体力
具体方法：1分钟不间断高强度运动,20秒钟休息.至少6个循环.

总结

如果你将要做有氧运动，并且希望自己的运动事倍功半。那么你就要在运动期间，将这9条建议牢记在心，以最大化地燃烧你的脂肪（并尽可能地保持你的肌肉）。​

1.每周至少做3次（依据个人目标增加次数），每次至少进行30分钟（最长40分钟搞定）的有氧训练。​

2.如果你对一个单独且连续的有氧训练感觉太过乏味，那么就把它分成较短的几段来进行。
（每段之间间隔不要超过10分钟）​

3.早晨或负重训练结束时，这是做有氧训练的最佳时机。​

4.对于燃脂的最好有氧方式不要忽略户外和需要调到全身的运动。​
​
5.在单纯有氧过程中（不要误解，这不代表你需要进行高强度或其他方式的有氧时的心率），将你的心率保持在你最大心率的70－80％区间（说话困难的时候），这能燃烧最多的脂肪。​

6.间隔式训练能使你的新陈代谢保持在一个较高的速率上，并在训练结束后仍然能够持续一段较长的时间，这能导致你大量地减少全身的体脂。

如何个性化你的Hexo博客

By Zhiyong Cai

发表于 2014-05-17

如何个性化你的Hexo博客

本人对排版和设计很看重，直接影响写博客的心情，所以纠结研究了很久。
经验总结成本文，希望对大家有帮助！

1.克隆喜欢的主题

复制主题

$ git clone https://github.com/wuchong/jacman.git themes/jacman

$ git clone https://github.com/A-limon/pacman.git themes/pacman

（也可以直接下载，复制到 themes目录下即可）

启用主题

修改Hexo根目录下的config.yml配置文件中的theme属性，将其设置为对应的主题名，如 jacman。

theme: jacman

本地调试

$ hexo clean
$ hexo g
$ hexo s

更新主题

$ cd themes/jacman
$ git pull

2.插件安装

多说评论功能

填写duoshuo_shortname多说的用户名，启用多说评论系统。在大陆地区更好用的评论系统

shortname为（注册多说账号之后，你的子域名就是了，比如xxx.duoshuo.com，那么xxx就是shortname）

启用微博秀

要启用微博秀，您必须填上author属性下tsina(必须是数字)和weibo_verifier（http://app.weibo.com/tool/weiboshow）的值

Q:为什么开启了微博秀后，显示是空白的，没有内容展示？
A:每次修改参数都会这样，需要多刷新几次或者上传到服务器上就好了

首页显示模式 Index

目前首页的显示模式支持两种，一种是原先的卡片式（前往 Demo 预览），另一种是类似官方主题的文章展开式（本站即采用的这种）。主题默认采用后一种展开式，如需开启第一种卡片式，请设置index属性下的expand: false。

目录 toc

是否启用在文章中或侧边栏中的目录功能。二者可以都为true或都为false。同时，如果你希望在特定的某一篇文章中关闭目录功能你可以在文章文件开头中的front-matter中加上一行toc: false。如果希望在某一篇文章中关闭目录自动编号功能你可以在文章文件开头的front-matter中加上一行list_number: false。

如何搭建你的Hexo博客

By Zhiyong Cai

发表于 2014-05-17

本文可以说是目前网上最为准确的Hexo博客搭建教程，为什么说最为准确呢？因为hexo博客搭建的命令时常有更新，按照网上旧有的教程会出很多问题，导致心情沮丧，当然我也是经历了这一过程，层层解决问题后把最优的方案呈现给大家，希望能帮到大家！经常会检查更新本文命令

1. 安装必备软件

1.1 Git
1.2 Node.js (推荐64位系统也安装32位，较少出错)

2. 注册Github

很简单，实在不懂可以百度

3. github上建立仓库

在github首页，点击页面右下角「New Repository」
project name：username.github.io
注意：Github Pages Repository名字必须是 username.github.io

4. 配置SSH keys

4.1 生成 SSH keys

$ ssh-keygen -t rsa -C "邮件地址@youremail.com"
Generating public/private rsa key pair.
Enter file in which to save the key (/Users/your_user_directory/.ssh/id_rsa):<回车就好>
Enter passphrase (empty for no passphrase):<回车就好>
Enter same passphrase again:<回车就好>

出现以下提示即生成SSH keys 成功

4.2 配置github SSH keys

1、搜索id_rsa.pub这个文件。用记事本打开，全选复制。
2、登陆github系统。点击右上角的 Account Settings—->SSH Public keys —-> add another public keys
3、把你本地生成的密钥复制到里面（key文本框中）， 点击 add key 就ok了

4.3 测试是否成功

测试命令

$ ssh -T git@github.com

如果出现

Are you sure you want to continue connecting (yes/no)?

输入yes即可，最后出现以下提示即配置成功

Hi caizhiyong! You've successfully authenticated, but GitHub does not provide shell access.

5. 安装Hexo(已更新，官方变动)

5.1 安装Hexo

以官方为准，参考【2】

$ npm install hexo-cli -g

5.2 部署Hexo

在任意磁盘新建文件夹Hexo
以D盘为例
进入Hexo文件夹内，再右键运行Git Bash
运行以下命令

$ hexo init
$ npm install

5.3 本地调试

$ hexo g <生成页面>
$ hexo s <搭建本地服务器页面>

在浏览器输入localhost:4000查看

6. 部署到Github（已更新，官方有变动）

6.1 _config.yml配置

配置Hexo文件夹下的 _config.yml文件

【旧】以前的方法

deploy:
type: github
repository: git@github.com:A-limon/alimon.github.com.git
branch: master

【更新】

deploy:
type: git
repository: git@github.com:onfeat/onfeat.github.com.git
branch: master

注意
type: github修改为git
repository: 一定要填写SSH连接，不要填https连接，容易出差错
： 后面都有一个空格，漏掉会出错

6.2 安装并测试

再运行以下命令：

$ npm install hexo-deployer-git --save
$ hexo g
$ hexo d

测试：
部署完成后，在浏览器中打开 http://username.github.io/，
网页能打开博客，部署成功。

7. 总结

7.1 部署总结

$ hexo clean
$ hexo generate
$ hexo deploy

7.2 本地调试

$ hexo g #生成
$ hexo s #启动本地服务，进行文章预览调试

浏览器输入localhost:4000，查看搭建效果。此后的每次变更_config.yml 文件或者新建文件都可以先用此命令调试，尤其是当你想调试新添加的主题时。

此命令可简化为：

$ hexo s -g

7.3 命令总结

常用命令

hexo new "postName" #新建文章
hexo new page "pageName" #新建页面
hexo generate #生成静态页面至public目录
hexo server #开启预览访问端口（默认端口4000，'ctrl + c'关闭server）
hexo deploy #将.deploy目录部署到GitHub
hexo help  # 查看帮助
hexo version  #查看Hexo的版本

复合命令

hexo deploy -g  #生成加部署
hexo server -g  #生成加预览

命令的简写为：

hexo n == hexo new
hexo g == hexo generate
hexo s == hexo server
hexo d == hexo deploy
